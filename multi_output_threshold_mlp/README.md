# Multi-Output Threshold MLP

This folder now keeps only the active player-points pipeline:

- `multi_output_threshold_mlp.py`
  - Stage 1 model.
  - Trains a multi-output threshold MLP on player-game box score features.
  - Scores deduped sportsbook player-line entries with both `Over` and `Under`.
- `stacked_bet_quality_model.py`
  - Stage 2 trust model.
  - Learns which positive-EV Stage 1 bets are worth keeping.
- `odds_processing.py`
  - Shared bookmaker-priority dedup and alternate-line parsing.
  - Used by both Stage 1 and the live app scoring path.
- `selection.py`
  - Shared same-team competition-aware selection logic.
  - Used by walk-forward evaluation instead of archived evaluator code.
- `walk_forward_stacked_backtest.py`
  - Monthly walk-forward evaluation for Stage 2.
  - The default selection rule is now the main production-style rule:
    decimal odds in `[2.0, 5.0)` and `trust_blend_score > 0.15`.
- `report_predictions.py`
  - Generates diagnostics on sides, bookmakers, probability bands, and dedup.

Older validation bundles, one-off evaluation scripts, and exploratory branches
belong in `archive/`.

## Main Decisions

- `Under`s stay in the model.
- Duplicate bookmaker copies of the same `(event, player, line)` are collapsed
  with bookmaker priority.
- The main downstream rule is:
  - `bet_source = stacked`
  - `2.0 <= bet_price < 5.0`
  - `trust_blend_score > 0.15`

## Core Outputs

- `multi_output_threshold_mlp_predictions.csv`
  - Stage 1 scored sportsbook entries.
- `stacked_bet_quality_predictions.csv`
  - Stage 2 scored entries with trust features.
- `walk_forward_stacked_selected_bets.csv`
  - Bets selected by the default walk-forward rule.
- `walk_forward_stacked_period_summary.csv`
  - Monthly summary for the default walk-forward rule.
- `diagnostics/`
  - Side, bookmaker, dedup, and probability-band summaries generated by
    `report_predictions.py`.

## Run Order

```bash
python3 multi_output_threshold_mlp/multi_output_threshold_mlp.py
python3 multi_output_threshold_mlp/stacked_bet_quality_model.py
python3 multi_output_threshold_mlp/walk_forward_stacked_backtest.py
python3 multi_output_threshold_mlp/report_predictions.py
```

The backtest defaults already use the current main rule, so extra flags are not
required unless you want to test a different threshold or odds band.
